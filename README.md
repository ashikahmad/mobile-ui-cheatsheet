# iOS ↔ Android UI Component Cheatsheet

A single-page reference mapping common mobile UI concepts to their iOS and Android platform terminology — for the moment a designer says "add a bottom sheet" and the iOS dev hears *Action Sheet* while the Android dev hears *BottomSheetDialog*.

**Live page:** https://ashikahmad.github.io/mobile-ui-cheatsheet/

## What's in it

Each entry covers a UI concept (e.g. "Circular primary-action button") and shows:

- The iOS term and framework API (UIKit / SwiftUI)
- The Android term and framework API (Views / Jetpack Compose)
- A short description of the pattern
- An expandable side-by-side mock UI preview of how it typically looks on each platform

Entries are grouped into categories — Navigation & Structure, Buttons & Actions, Input & Selection, Lists & Collections, Overlays & Sheets, Feedback & Status, System Chrome & Layout, and Design Language & Frameworks — and are searchable/filterable from the page itself.

## Contributing

Still no build step — just plain static files served as-is. Component data lives in `data.js`; mock UI previews are generated by the renderers in `mocks.js`; page behavior (search, sidebar navigation, the mobile drawer) lives in `app.js`; everything visual is in `styles.css`. Edit whichever file matches your change and open a PR — spotted an inaccuracy or missing component? Fixes welcome.

## Deployment

Served as-is via GitHub Pages from the `main` branch root.
